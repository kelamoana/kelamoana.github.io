PetrChat Server

C

Designed a multi-threaded server that could concurrently host hundreds of users that had the capability to create/delete chat rooms, send direct messages, and customize a username.


A New Approach to Coding:

Due to the density of this project, it was apparent that it would be easy to get lost with concurrency bugs. With that in mind, my partner and I decided to design the entire project from head to toe, before we typed any code. We discussed tradeoffs of each potential design decision on micro and macro levels. This led to our success with the project - only running into 2 quick-fix concurrency bugs and finishing with a robust and efficient application capable of holding a large number of users.

Custom Linux Shell

C

Developed a shell that allowed a user to change directories, read/write to files, and pipe commands.


What I learned:

By creating this shell I learned the intricacies of process forking and the different challenges that comes with implementing it in a program, especially one with the capability of piping/directing input and output. Keeping track of PIDs, properly reaping zombie children, managing memory, and understanding the scheduling process were the most difficult aspects of creating the logic for the program. By whiteboarding my code design and learning to create small python scripts capable of giving input to the shell and writing to stderror, I was able to incrementally develop and test the program.
